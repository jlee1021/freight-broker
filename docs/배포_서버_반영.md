# 서버 반영 (우선순위 높음 기능 배포)

로컬에서 수정한 코드를 **Ubuntu 서버**에 반영할 때 순서대로 실행하세요.

---

## 1. 코드 서버에 가져오기

로컬에서 git으로 푸시했다면 서버에서 pull:

```bash
cd ~/freight-broker
git pull
```

git을 쓰지 않으면, 수정된 파일을 **rsync/scp** 등으로 서버에 복사하세요.  
최소 반영 대상:
- `backend/` (마이그레이션 010, 모델·API·스키마 변경)
- `frontend/` (App, AuthContext, Account, Invoicing, Reports 등)

---

## 2. 백엔드 마이그레이션 실행

```bash
cd ~/freight-broker/backend
source .venv/bin/activate
export PYTHONPATH=.
python scripts/run_migrate.py
```

`Migration completed: upgrade head` 가 나오면 010(users.partner_id) 적용된 것입니다.

---

## 3. 백엔드 재시작

현재 uvicorn을 **수동**으로 띄운 경우:
- 해당 터미널에서 `Ctrl+C` 후 다시:

```bash
cd ~/freight-broker/backend
source .venv/bin/activate
uvicorn app.main:app --host 0.0.0.0 --port 8001
```

**systemd**로 서비스 등록해 두었다면:

```bash
sudo systemctl restart freight-broker-api
# 또는 사용 중인 서비스 이름에 맞게
```

---

## 4. 프론트엔드 반영 (배포 빌드 사용 시)

프론트를 **빌드**해서 nginx 등으로 서빙한다면:

```bash
cd ~/freight-broker/frontend
npm ci
npm run build
```

이후 웹서버가 `dist` 를 바라보고 있으면 새로고침만 하면 됩니다.

개발 서버(`npm run dev`)만 쓴다면, 코드 반영 후 해당 프로세스만 다시 띄우면 됩니다.

---

## 5. 확인

- 브라우저에서 로그인 후:
  - **Account** → 사용자 추가/수정 시 "Portal partner" 선택 가능한지
  - **Invoicing** → 연체 인보이스 행에 "Send reminder" 버튼 있는지
  - **Reports** → "By Lane (Origin → Destination)" 테이블 보이는지
- 포털: Account에서 특정 사용자에 고객/캐리어 Partner 연결 후, 그 계정으로 로그인하면 "My Loads", "My Invoices"만 보이는지

---

*마이그레이션 실패 시: `.env` 의 `DATABASE_URL`(예: localhost:5433)과 PostgreSQL 실행 여부를 확인하세요.*
