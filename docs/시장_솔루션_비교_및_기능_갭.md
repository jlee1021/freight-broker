## 2. 시장 공통 핵심 기능 (기능군별)

### 2.1 로드(화물) 관리
- 로드 생성·수정·상태 관리 (pending → assigned → in transit → delivered 등)
- **실시간 로드 리스트**·대시보드(한 화면에서 상태 변경·할당)
- **견적(Quote)·Rate Confirmation** 자동/반자동 생성
- Shipper/Consignee/스탑 정보, 픽업·딜리버리 날짜/시간
- **로드보드 연동** — 여러 로드보드에 한 번에 포스팅 (Dat, Truckstop 등)
- Equipment type, weight, commodity 등 화물 스펙

### 2.2 캐리어(운송사) 관리
- 캐리어 DB (연락처, MC/DOT, 보험 만료일)
- **캐리어 성과·평점** (온타임률, 신뢰도)
- **캐리어 매칭** — 레인·가용성·위치 기반 검색
- **캐리어 온보딩·벤딩** — 보험·서류 자동 갱신·사기 탐지
- Tender(화물 제안) 발송·수락/거절

### 2.3 문서 관리
- **Rate Confirmation (RC)**, **BOL**, **Pallet Tag** 등 **자동 생성·PDF**
- **POD(Proof of Delivery)** 업로드·로드/캐리어별 보관
- 문서를 고객/캐리어/로드별로 연결해 검색·다운로드
- 회사 로고·브랜딩이 들어간 문서

### 2.4 재무·인보이스
- **고객 인보이스(AR)** — 자동/일괄 생성, 상태(draft/sent/paid)
- **캐리어 청구(AP)** — 캐리어 빌 매칭, RC와 대조·차이 플래그
- **회계 연동** — QuickBooks, Xero 등과 동기화
- **결제 처리** — Epay, RoadSync, TriumphPay 등 연동
- 마진·수익률 리포트 (로드/고객/캐리어별)

### 2.5 디스패치·가시성
- **실시간 트래킹** — Macropoint, TruckerTools, TextLocate 등 연동
- **체크콜** — 일괄 체크콜(한 번에 여러 로드)
- 로드 상태 업데이트 (픽업 완료, 인트랜싷, 딜리버드)
- 고객/캐리어 **포털** — 문서 조회·업로드·상태 확인

### 2.6 자동화·연동
- 반복 작업 자동화 (로드 정보·캐리어 정보 자동 채우기)
- **EDI·API** — 고객/캐리어/로드보드와 데이터 교환
- 이메일/SMTP로 RC·BOL·인보이스 발송
- 알림·리마인더 (보험 만료, 미결제 인보이스 등)

### 2.7 리포팅·대시보드
- **마진·수익** 대시보드 (고객/레인/캐리어별)
- 캐리어 온타임률·성과 트렌드
- AR/AP 현황·미수금
- 커스텀 리포트·필터·엑셀 내보내기

## 3. 우리 솔루션 vs 시장 기능 비교

| 기능 영역 | 시장 보통 기능 | 우리 프로젝트 현황 | 갭(보강 시 우선순위 참고) |
|-----------|----------------|-------------------|---------------------------|
| **로드 관리** | 로드 CRUD, 상태, RC/BOL 생성, 로드보드 포스팅 | ✅ 로드 CRUD, 상태, 스탑·캐리어 세그먼트, RC/BOL/PDF | ⬜ 로드보드 연동 없음 |
| **캐리어 관리** | DB, 성과·평점, 매칭, 벤딩·보험 갱신 | ✅ 파트너(캐리어) DB, MC/DOT/보험만료 | ⬜ 캐리어 평점·매칭·벤딩 연동 없음 |
| **문서** | RC, BOL, POD, 브랜딩 PDF | ✅ RC, BOL, Pallet Tag, PDF, 이메일 발송 | ⬜ POD를 로드별로 더 강하게 연결·검색, 브랜딩 설정은 제한적 |
| **재무** | AR/AP, 회계 연동, 결제 연동 | ✅ AR/AP 인보이스, 상태, PDF | ⬜ QuickBooks 등 회계 연동 없음, 결제 연동 없음 |
| **트래킹·디스패치** | 실시간 위치, 체크콜, 상태 업데이트 | ✅ 로드 상태 수동 변경, 로드 노트 | ⬜ 실시간 트래킹 연동 없음, 일괄 체크콜 없음 |
| **포털** | 고객/캐리어 셀프서비스 | ❌ 없음 | ⬜ 고객·캐리어 포털 없음 |
| **자동화·연동** | EDI, API, 자동 채우기 | ✅ API 존재, SMTP 이메일 | ⬜ EDI·로드보드 API 연동 없음 |
| **리포팅** | 마진/수익 대시보드, 온타임률, AR/AP | ✅ Dashboard, Profit, Reports, Invoicing | ⬜ 레인/캐리어별 고급 분석·엑셀 내보내기 등 보강 여지 |
| **권한·설정** | 역할별 메뉴, 회사 정보 | ✅ admin/dispatcher 등 역할, Setting | ✅ 상대적으로 충족 |

## 4. 기능 완성도 높이기 — 추천 우선순위

우리 앱에 **이미 있는 것**: 로드·파트너(고객/캐리어)·문서(RC/BOL/PDF)·AR/AP·대시보드·Profit·Reports·권한·Setting·Inventory·로드 노트·첨부파일.

**기능적으로 더 완성도를 높이려면** 아래 순서를 추천합니다.

### 높음 (실무에서 바로 체감)
1. **로드별 POD(Proof of Delivery) 명확화** — 이미 첨부는 있음. “POD” 타입/필터·문서 목록에서 POD 강조하면 문서 관리가 시장 수준에 가까워짐.
2. **캐리어 성과·평점** — 파트너(캐리어)에 평점·온타임 여부 저장 + 로드와 연결. 리포트에 “캐리어별 온타임률” 추가.
3. **리포트 보강** — 레인(구간)별·고객별·캐리어별 마진/수익, AR/AP 요약. 필요 시 **CSV/Excel 내보내기**.
4. **고객 인보이스 due date·리마인더** — Setting에 “미수금 X일 경과 시 알림” 같은 간단 규칙 또는 리스트 필터.

### 중간 (차별화·운영 효율)
5. **회계 연동** — QuickBooks 등과 AR/AP 동기화(API 또는 CSV 내보내기).
6. **문서 브랜딩** — RC/BOL PDF에 Setting의 회사명·로고 URL 반영 (이미 회사명은 부분 반영 가능).
7. **일괄 체크콜/상태 업데이트** — 여러 로드 선택 후 “상태 일괄 변경” 또는 “체크콜 필요” 표시.
8. **알림·리마인더** — 보험 만료 임박 파트너, 미결제 인보이스 등 (이메일 또는 대시보드 알림).

### 낮음 (장기·연동)
9. **로드보드 연동** — Dat, Truckstop 등 API로 로드 포스팅(별도 API 키·비용).
10. **실시간 트래킹** — Macropoint, TruckerTools 등 연동(별도 비용·계약).
11. **고객/캐리어 포털** — 로그인한 고객·캐리어가 자기 로드/문서만 보는 화면.
