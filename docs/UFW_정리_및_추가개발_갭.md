# UFW 포트 정리 + 시장 대비 추가 개발 갭

## 1. UFW 포트 정리

### 현재 열린 포트 (당신이 보낸 상태 기준)

| 포트      | 용도                    | 권장 |
|-----------|-------------------------|------|
| OpenSSH   | SSH 접속                 | **유지** (필수) |
| 36888/tcp | (용도 불명)             | 본인 서비스 아니면 **삭제** |
| 36900/tcp | (용도 불명)             | 본인 서비스 아니면 **삭제** |
| 8025/tcp  | (용도 불명, 메일 관련?) | 본인 서비스 아니면 **삭제** |
| 1025/tcp  | (용도 불명)             | 본인 서비스 아니면 **삭제** |
| 8000/tcp  | Freight Broker API (원래) | **삭제** (실제 사용은 8001) |
| 5173/tcp  | Freight Broker Frontend | **유지** |
| 8001/tcp  | Freight Broker API (현재 사용) | **유지** |

### Freight Broker만 쓸 때 최소 규칙

- **필수:** OpenSSH, 5173, 8001  
- **제거해도 되는 것:** 8000 (API는 8001만 사용)

### 삭제할 때 명령 (예시)

```bash
# 8000 삭제 (API는 8001만 사용하므로)
sudo ufw delete allow 8000/tcp

# 36888, 36900, 8025, 1025 — 다른 서비스에서 안 쓰면 삭제
sudo ufw delete allow 36888/tcp
sudo ufw delete allow 36900/tcp
sudo ufw delete allow 8025/tcp
sudo ufw delete allow 1025/tcp
```

**주의:** 36888, 36900, 8025, 1025는 **다른 앱(메일, 모니터링 등)에서 쓰는지 먼저 확인**한 뒤, 안 쓰면 위처럼 삭제하세요.

```bash
sudo ufw reload
sudo ufw status
```

## 2. 시장 솔루션 대비 — 이미 구현된 것

우리 프로젝트에서 **이미 반영된 기능** (문서·대화 기준):

- 로드 CRUD, 상태, 스탑, 캐리어 세그먼트
- Rate Confirmation / BOL / Pallet Tag 생성·PDF·이메일 발송
- **문서 브랜딩** (Setting 회사명·로고 URL → RC/BOL 등 상단 표시)
- **POD 명확화** (첨부 document_type: POD/BOL/RC 등, 필터·표시)
- **캐리어 성과** (평점 1–5, 온타임 여부, Reports 캐리어 성과 테이블)
- **일괄 상태 변경** (Order 목록 체크박스 → PATCH bulk-status)
- AR/AP 인보이스, 상태, PDF, **연체 필터(Overdue)**·Due 컬럼
- **대시보드 알림** (연체 인보이스 수, 보험 만료 임박 캐리어 수)
- **리포트·CSV** (Export loads CSV, AR CSV, AP CSV, Revenue 리포트)
- Setting (회사명·로고 URL·SMTP 등), 역할(admin/dispatcher), Inventory

## 3. 시장 솔루션 대비 — 추가로 하면 좋은 것

시장 조사(BrokerPro, ARK TMS, 2025 Buyer's Guide 등)와 `시장_솔루션_비교_및_기능_갭.md` 기준으로, **아직 없거나 보강 여지가 있는 것**만 정리했습니다.

### 3.1 우선순위 높음 (실무 체감)

| 항목 | 내용 | 시장 참고 |
|------|------|-----------|
| **고객/캐리어 포털** | 고객·캐리어가 로그인해 “내 로드·내 문서·내 인보이스”만 보는 전용 화면 | Carrier/Shipper Portal, 셀프서비스 문서·결제 조회 |
| **인보이스 리마인더 자동화** | due 경과 시 이메일 리마인더 또는 대시보드 “미수금 리마인더” 목록 | 미결제 인보이스 알림·자동 리마인더 |
| **레인(구간)별 리포트** | Lane/Origin–Destination별 수익·마진·건수 | 레인별 분석, 레인 가격 트렌드 |

### 3.2 우선순위 중간 (차별화·운영 효율)

| 항목 | 내용 | 시장 참고 |
|------|------|-----------|
| **회계 연동** | QuickBooks / Xero 등과 AR·AP 동기화 (API 또는 CSV 업로드) | QuickBooks Sync, 인보이스·결제 실시간 동기화 |
| **결제 연동** | Epay, RoadSync, TriumphPay 등 캐리어 결제 처리 | AP 자동 결제·리콘실리에이션 |
| **Tender(화물 제안) 발송** | 로드 정보를 캐리어에게 이메일/앱으로 제안 → 수락/거절 | Tender 발송·수락·거절 워크플로우 |
| **캐리어 매칭·검색** | 레인·가용성·위치·평점 기반 캐리어 검색/필터 | 캐리어 DB 검색·매칭 |
| **문서 이메일 자동화** | RC/BOL 발송 시 “자동 이메일” 옵션 또는 워크플로 규칙 | RC·BOL·POD 자동 발송 |

### 3.3 우선순위 낮음 (장기·외부 연동)

| 항목 | 내용 | 시장 참고 |
|------|------|-----------|
| **로드보드 연동** | Dat, Truckstop 등 API로 로드 포스팅 | 로드보드 포스팅·가시성 |
| **실시간 트래킹** | Macropoint, TruckerTools 등 GPS/체크콜 연동 | 실시간 위치·트래킹 |
| **EDI·고객/캐리어 API** | EDI 또는 REST API로 주문·상태·문서 교환 | EDI·API 연동 |
| **전자 서명** | RC/BOL·POD 등에 드라이버/고객 전자 서명 | 모바일 전자 서명 |

## 4. 요약

- **UFW:** Freight Broker만 쓴다면 **OpenSSH + 5173 + 8001** 만 열고, **8000은 삭제**. 36888/36900/8025/1025는 다른 서비스에서 안 쓰면 삭제 검토.
- **추가 개발:** 시장 대비해서 **고객/캐리어 포털**, **인보이스 리마인더**, **레인별 리포트**를 먼저 넣고, 그다음 **회계(QuickBooks) 연동**, **Tender·캐리어 매칭** 순으로 가면 실무 체감이 큽니다.
